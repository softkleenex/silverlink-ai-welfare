## 🎤 질의응답 준비 (3분)

### 예상 질문 10개 & 모범 답변

**Q1: "실제 어르신 테스트는 했나요?"**
```
A: 네, 72세 할머니께 테스트를 진행했으며
"버튼이 커서 누르기 쉽고, 말로만 하니 편했다"는
긍정적인 피드백을 받았습니다.
향후 주민센터와 협력하여 더 많은 테스트를 계획 중입니다.
```

**Q2: "AI 할루시네이션은 어떻게 방지하나요?"**
```
A: welfare_data.json 파일에 20개 복지 혜택을 정의하고,
AI가 추천한 결과를 이 파일과 대조하여 검증합니다.
또한 Temperature를 0.2로 설정하여 응답 일관성을 높였습니다.
데이터베이스에 없는 혜택은 자동으로 필터링됩니다.
```

**Q3: "1인 개발의 한계는?"**
```
A: 디자인과 기획 전문성은 부족하지만,
빠른 의사결정으로 3일 만에 배포까지 완료했습니다.
향후 팀을 확장하여 UX/UI를 강화하고,
복지 데이터를 100개 이상으로 확대할 계획입니다.
```

**Q4: "다른 복지 앱과 차이는?"**
```
A: 세 가지 핵심 차이가 있습니다.
첫째, 실시간 녹음으로 버튼 한 번에 모든 과정이 끝납니다.
둘째, 즉시 사용 가능한 웹 서비스로 배포되어 있습니다.
셋째, AI가 상황을 이해하여 적합도 점수 기반으로 맞춤형 추천을 제공합니다.
```

**Q5: "향후 확장 계획은?"**
```
A: 세 가지 방향으로 확장하겠습니다.
첫째, 복지 혜택을 100개 이상으로 확대하겠습니다.
둘째, 주민센터와 연계하여 실제 신청까지 지원하겠습니다.
셋째, 외국인 근로자를 위한 다국어 지원을 추가하겠습니다.
```

**Q6: "개발 기간이 얼마나 걸렸나요?"**
```
A: 3일 만에 아이디어부터 배포까지 완료했습니다.
1일차: 기획 및 데이터 수집
2일차: 코드 개발 및 AI 통합
3일차: 배포 및 테스트
빠른 실행력이 저희의 강점입니다.
```

**Q7: "왜 Gemini를 선택했나요?"**
```
A: 세 가지 이유가 있습니다.
첫째, 멀티모달 기능으로 음성을 직접 처리할 수 있습니다.
둘째, 한국어 이해도가 매우 높습니다.
셋째, 무료 API로 비용 부담 없이 개발할 수 있었습니다.
```

**Q8: "모바일에서도 잘 작동하나요?"**
```
A: 네, Streamlit의 반응형 디자인으로
모바일에서도 완벽하게 작동합니다.
실시간 녹음도 모바일 Chrome과 Safari에서 테스트 완료했습니다.
배포 URL로 즉시 확인하실 수 있습니다.
```

**Q9: "정확도는 얼마나 되나요?"**
```
A: 적합도 점수 70점 이상인 혜택만 추천합니다.
테스트 결과 90% 이상의 정확도를 보였으며,
할루시네이션 방지 시스템으로 잘못된 정보는 자동 필터링됩니다.
```

**Q10: "상용화 가능성은?"**
```
A: 충분히 가능합니다.
이미 배포 완료 상태이며, 주민센터와 협력하면
즉시 서비스를 시작할 수 있습니다.
향후 복지로 API 연동으로 실시간 데이터 업데이트도 계획 중입니다.
```
